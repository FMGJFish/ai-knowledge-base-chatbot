# Architecture Reference

Per ADR Decision 001 (Project Repository Strategy), this repository is authoritative for source code, build configuration, deployment assets, and runtime documentation. It is **not** the authoritative source for architecture or governance.

The authoritative architecture and governance documents live in the Fisher AI Automation Portfolio Development Program (FAAPDP) governance repository, under `docs/reference_implementations/project_01_ai_knowledge_base_chatbot/`:

- `01_business_requirements_v1.md` — Business Requirements
- `02_system_architecture_v1.md` — System Architecture
- `03_database_design_v1.md` — Database Design
- `04_api_design_v1.md` — API Design
- `05_security_review_v1.md` — Security Review
- `06_deployment_architecture_v1.md` — Deployment Architecture
- `07_operational_architecture_v1.md` — Operational Architecture
- `implementation_assumptions_v1.md` — Implementation Assumptions
- `architectural_decision_record_v1.md` — Architectural Decision Record
- `technical_specification_v1.md` — Technical Specification
- `implementation_roadmap_v1.md` — Implementation Roadmap
- `project_status_v1.md` — Project Status

Program-wide governance (engineering standards, technology stack, documentation standards, development lifecycle, definition of done, team roles, implementation protocol) lives under `docs/governance/` in the same repository.

Runtime documentation specific to this implementation (installation, configuration, running the project) lives in this repository's `README.md`.
